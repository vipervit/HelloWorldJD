A simple hello-world project to demonstrate at the most basic level how to make a Jenkins pipeline and integrate it with Docker and PyPI for continous delivery. 

 

Note that in case of this project there is no continuous demployment as the end user will have to install the latest version of the "product" (i.e. either a Docker image or a Python package or both) manually.

Here is a good read for understanding the difference between the terms (googling "continous delivery vs.." will also help): Atlassian CI/CD

 

Docker image

Docker image of the demo is available from Docker Hub:  https://hub.docker.com/r/tesqos/demojd

To install the image: docker pull tesqos/demojd

To run it upon the installation: docker run tesqow/demojd

To check it in the local Docker repository: docker image ls

Python package

The same product in the form of a Python package can be downloaded and install on the local machine from the PyPI index using the following command:

pip install demojd

To run the code: python -m demojd.scripts.hello

 

The result of the execution produces output from different modules in the package tree and looks as follows:

Hi from uncle!

Hi from mom!

Hi from dad!

Hi from son!

Hi from daughter!

Hi from cousin!

Hi from niece!

Hi from nephew!

Hi from grandma!

 

Software Modification Process

Let's demonstrate modification of the project's code by introducing changes to a data file called otherslist.json.

In order to do that, anyone having a GitHub account can propose changes to the file by editing it (in their forked branch) and create a pull request. 

Once the maintainer of the project incorporates the change in a new version (thus making the user proposing the changes a contributor to the project), the change becomes part of the new code.

Pushing the changes to the Github repository will generate a new build and ultimately new versions of the executables.

For example, if we modify the otherlist.json file by adding a new line 'grandpa', the software output will grow by a new line 'Hi from grandpa!' because a function reads the data file and sends a formatted output to stdout using all the lines of the file.
